# Quick Start Guide

Follow these steps to get Nebula Backend up and running on your local machine.

## Prerequisites

<CardGroup cols={2}>
  <Card title="Go 1.18+" icon="golang">
    Required for local development
  </Card>
  <Card title="Docker" icon="docker">
    Required for containerized deployment
  </Card>
</CardGroup>

## Installation

<Steps>
  <Step title="Clone the Repository">
    ```bash
    git clone https://github.com/Annany2002/nebula-backend.git
    cd nebula-backend
    ```
  </Step>
  
  <Step title="Configure Environment">
    Copy the example environment file and configure it:
    
    ```bash
    cp .env.example .env
    ```
    
    Edit `.env` with your settings:
    
    ```bash
    # CRITICAL: Set a strong, unique secret
    JWT_SECRET=your-super-secret-key-change-this
    
    # Server configuration
    SERVER_PORT=8080
    
    # CORS origins (space-separated)
    ALLOWED_ORIGINS=http://localhost:3000 http://localhost:5173
    
    # JWT expiration in hours
    JWT_EXPIRATION_HOURS=24
    ```
    
    <Warning>
      Never commit your `.env` file to version control. Always use a strong, unique `JWT_SECRET` in production.
    </Warning>
  </Step>
  
  <Step title="Run with Docker Compose">
    Start all services with a single command:
    
    ```bash
    docker-compose up --build
    ```
    
    This starts:
    - **Backend API** on port `8080`
    - **Documentation** on port `3000`
  </Step>
</Steps>

## Verify Installation

Test that the API is running:

<CodeGroup>
```bash cURL
curl http://localhost:8080/ping
```

```bash Expected Response
pong
```
</CodeGroup>

Check the health endpoint:

```bash
curl http://localhost:8080/health
# Returns: 200 OK
```
